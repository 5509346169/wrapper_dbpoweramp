# Documentation site

The Hugo project that builds and deploys the dBpoweramp Wrapper documentation site lives entirely in this directory.

## Layout

- `hugo.yaml` — Hugo configuration (baseURL, title, outputs, markup)
- `data/navigation.yaml` — hub-and-spoke nav tree consumed by the sidebar
- `data/presets.yaml` — preset metadata for codec chips and the landing cards
- `data/versions.yaml` — release data for the version selector
- `themes/hugo-audiophile/assets/scss/` — design tokens + component styles (Sass partials)
- `themes/hugo-audiophile/layouts/` — `default`, `landing`, `reference`, `print` layouts
- `themes/hugo-audiophile/layouts/partials/` — shell partials (head, header, sidebar, footer, right-rail, command-palette)
- `themes/hugo-audiophile/layouts/shortcodes/` — components (waveform, codec-chip, callout, code-block, mermaid, graphviz, version-selector)
- `assets/js/` — source ES modules, bundled to `assets/vendor/` by `npm run vendor`
- `assets/vendor/` — locally bundled JS (no CDN), generated by `npm run vendor`
- `scripts/vendor-build.mjs` — esbuild entry that bundles the JS modules
- `scripts/build-css.mjs` — Dart Sass compiler (compiles theme SCSS → `static/css/main.css`)
- `scripts/check-no-cdn.mjs` — CI guard that fails the build if a CDN URL slips into `public/`
- `content/` — Markdown pages, organised by Hugo section (`getting-started/`, `configuration/`, `architecture/`, `engineering/`, `reference/`)
- `static/_redirects` — Jekyll → Hugo URL redirects (GitHub Pages format)
- `package.json` — Node dependencies (esbuild, sass, cpx2, rimraf)

## Local dev

```sh
cd _docs_site
npm install
npm run css && npm run vendor
hugo server
```

Hugo serves the site at <http://localhost:1313/wrapper_dbpoweramp/>.

## Build a deployable artifact

```sh
cd _docs_site
npm run css && npm run vendor
hugo --gc --minify --environment production
node scripts/check-no-cdn.mjs public
```

Hugo outputs to `public/`. `static/` files are copied into `public/` at build time.

## Hugo version

Pin to **Hugo 0.139** (extended). Install via the [Hugo releases page](https://github.com/gohugoio/hugo/releases/tag/v0.139.0) or `brew install hugo` on macOS.
