# Documentation site

The Jekyll project that builds and deploys the dBpoweramp Wrapper documentation site lives entirely in this directory.

## Layout

- `_config.yml` — Jekyll configuration (title, URL, plugins, theme tokens)
- `_data/navigation.yml` — hub-and-spoke nav tree consumed by the sidebar
- `_data/presets.yml` — preset metadata for codec chips and the landing cards
- `_data/versions.yml` — release data for the version selector
- `_sass/` — design tokens + component styles, imported from `assets/css/main.scss`
- `_layouts/` — `default`, `landing`, `reference`, `print`
- `_includes/` — `layout/` (shell partials), `components/` (waveform, codec-chip, callout, code-block, mermaid, graphviz, version-selector, command-palette), `search/`
- `_plugins/` — custom Liquid tags (`mermaid`, `graphviz`, `toc_depth`)
- `assets/css/main.scss` — single import file (everything else is a `_sass/` partial)
- `assets/js/` — source ES modules, bundled to `assets/vendor/` by `npm run vendor`
- `assets/vendor/` — locally bundled JS (no CDN), generated by `scripts/vendor-build.mjs`
- `scripts/vendor-build.mjs` — esbuild entry that bundles the JS modules
- `scripts/check-no-cdn.mjs` — CI guard that fails the build if a CDN URL slips into `site/`
- `contents/` — Markdown pages, organised by navigation category; permalinks (`/installation/`, `/cli/`, etc.) preserve the URLs that the previous MkDocs site published
- `Gemfile` / `package.json` — Ruby and Node dependencies (gem + npm)
- `_site/` — Jekyll's intermediate build output (gitignored)
- `site/` — final deployable artifact, produced by `make deploy` or `rake deploy` (gitignored)

## Local dev

```sh
cd _docs_site
bundle install
npm install
make docs-serve
```

The first run will compile Sass, bundle the JS modules into `assets/vendor/`, and serve the site at <http://127.0.0.1:4000>.

## Build a deployable artifact

```sh
cd _docs_site
make docs-build   # writes _site/
make deploy       # copies _site/ -> site/
make check-cdn    # fails if site/ contains any CDN URL
```

## CI

`.github/workflows/deploy.yml` runs `bundle install`, `npm ci && npm run vendor`, `bundle exec jekyll build --strict_front_matter`, the CDN guard, and uploads `site/` to GitHub Pages.
